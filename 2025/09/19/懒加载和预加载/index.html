懒加载和预加载

发表于2025-09-19|更新于2025-09-23|前端性能优化懒加载和预加载
|浏览量:

概念

懒加载

在图片进入可视区之后再请求图片资源是一个很典型的懒加载case

懒加载适用于对于电商等图片很多，页面很长的业务场景适用

懒加载可以减少无效资源的加载，也能解决并发加载的资源过多造成的 JS 阻塞

预加载

和懒加载是一个相反的过程，是对资源的提前请求。在资源需要被使用时能从缓存中加载，提升用户体验。

使用预加载也能去维护一些页面展示的一个依赖关系。

比如大型图片组成的动画，抽奖组件等

懒加载的使用

以图片进入视口后再加载这个需求为例

HTML原生懒加载

loading = “lazy”

逐项解释

  • loading=”lazy”：告诉浏览器延迟加载图片，直到图片接近/进入视口。浏览器决定何时加载。
  • decoding=”async”：提醒浏览器异步解码图像，减少阻塞渲染
  • alt：可访问性必须选，屏幕阅读器依赖它

传统滚动监听

缺点：

  • scroll事件高频触发
  • 对复杂布局，iframes，快速滚动等场景不可靠

InterselectionObserver

由浏览器底层高效实现，不会像 scroll 那样频繁回调，也可以设置 rootMargin 实现预加载体验，而且清晰的 unobserve 机制能避免内存泄露

核心概念

new InterselectionObserver(callback, options)

  • callback(entries, observer)：当被观察元素与root交集状态发生变化时调用
  • entries：IntersectionObserverEntry 的数组，每项包含 isIntersecting、intersectionRatio、boundingClientRect、intersectionRect、rootBounds、target、time 等属性。

options：

  • root：根元素，默认为null，即视口
  • rootMargin：扩展/收缩 root 的矩形，与CSS中的margin类似
  • threshold：触发阈值，0（任意交叉触发），1.0（完全进入触发），也可传数组 [0, 0.5, 1]

预加载的使用

浏览器原生

1.<link rel="preload">

告诉浏览器在渲染之前就把图片资源请求下来

2.<link rel="prefetch">

低优先级下载，适合提前准备“未来页面”要用到的图片。

JS控制方式

1.使用 image 对象

2.用 fetch() + blob

XMLHttpRequest

不推荐

框架和工具链支持

1.React / Vue 中的图片预加载

  • React：用 Suspense + 资源预加载，或者直接在组件挂载前调用 new Image().src = url;。
  • Vue：常用 mixin 或 composition API 在 onMounted 前执行 Image() 预加载。

2.Next.js / Nuxt.js 等 SSR 框架

它们会自动为关键资源加上 <link rel="preload">，并且支持 next/image 这种组件，内置优化（懒加载、尺寸适配、webp 转换）。

Service Worker 缓存

概念

Service Worker是浏览器的一个后台脚本，它独立于页面运行，可以拦截网络请求、管理缓存

实现“预加载”的核心思路就是：在 SW 安装阶段（install），提前把某些资源缓存下来，这样用户访问时就能直接命中缓存。

如果是 PWA 或者单页应用，可以用 Service Worker 提前缓存图片：

当用户第一次访问时，SW 就会把关键资源预加载到本地缓存。下次再打开页面，甚至离线时，也能直接使用。
